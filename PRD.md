---

## 1. 배경 및 목적

### 배경
- 쿠팡 로켓그로스 셀러(AS Gel 전인수 대표님 외 다수)의 핵심 페인포인트는 **재고 관리**
- 기존 솔루션(트렌드온, 빅셀, 셀로 등)은 재고 기능이 부족하거나 구매대행/배대지 종속적
- 장사왕은 현재 재고 관련 기능 전무

### 목적
- 쿠팡 Wing 스크래핑 기반 **마켓재고 자동 수집**
- 입고권장 자동 계산으로 **발주 의사결정 지원**
- **POC → MVP → 고객 검증** 순서로 빠른 PMF 확인

### 목표 지표
- 시연 후 전인수 대표님 포함 베타 유저 3명 확보
- 주 3회 이상 재고 화면 접속 달성
- 입고권장 → 실제 발주 연결율 50% 이상

---

## 2. 타겟 유저

### Primary: 쿠팡 로켓그로스 셀러
- 상품 수: 30~200 SKU
- 직원 수: 1~5명
- 재고 관리: 현재 수동(엑셀, 쿠팡 직접 확인)
- 주요 고통: 품절로 인한 순위 하락, 과잉재고로 인한 보관비

### 유저 시나리오
```
매일 아침 → 재고 현황 확인 → 입고권장 확인 → 발주 여부 결정
주 1~2회 → 발주 입력 → 입고 프로세스 관리
```

---

## 3. MVP 범위

### ✅ MVP In (구현)
1. 쿠팡 Wing 마켓재고 자동 스크래핑
2. 상품관리 화면 (원가 이력, 마켓재고 팝업, 바코드 인쇄)
3. 재고관리 - 일일매출/재고 화면 (입고권장 계산)
4. 재고관리(주문) 프로세스 탭 (구매확정 ~ 상품수령완료)
5. 대시보드 캘린더 재고 항목 추가

### ❌ MVP Out (Phase 2 이후)
- 발주요청/발주금액확정 탭 (배대지 연동 필요)
- 광고 데이터 연동 (별도 스프린트)
- 배대지 자동 연동
- 자연판매/광고판매 구분 (쿠팡 데이터 제공 시)

### 4.3 재고관리 - 일일매출/재고

**테이블 컬럼**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| + | 버튼 | 옵션 펼치기 (옵션명/옵션코드) |
| 별칭 | 자동 | 로켓/그로스/윙 |
| 상품ID | 자동 | |
| 상품명 | 자동 | |
| 옵션명 | 자동 | + 펼치면 노출 |
| 순이익금 | 자동 계산 | 일일 순이익 |
| 재고금액 | 자동 계산 | 마켓재고 × 원가 |
| 월판매수량 | 스크래핑 | 오늘 기준 30일 |
| 일판매수량 | 스크래핑 | 당일 판매 수량 |
| 광고판매량 | 스크래핑 | MVP에서 0 표시 가능 |
| 자연판매량 | 스크래핑 | MVP에서 0 표시 가능 |
| 진행광고비 | 스크래핑 | MVP에서 0 표시 가능 |
| 마켓재고 | 스크래핑+수동 | |
| 국내총재고 | 수동 | |
| 일목표수량 | 수동 설정 | 월목표 ÷ 해당월 일수 |
| 입고권장 | 자동 계산 | 아래 로직 참고 |
| 입고중 | 주문 연동 | 재고관리(주문)에서 연동 |
| 재고입고 | 수동 | 입고 완료 후 수량 입력 |
| 재고주문 | 버튼 | 클릭 → 재고관리(주문) 구매확정 생성 |

**입고권장 계산 로직**
```
월목표수량 설정 O:
  일목표 = 월목표 ÷ 해당월 일수
  권장입고 = (일목표 × 남은일수) - 마켓재고 - 입고중

월목표수량 설정 X:
  일평균 = 월판매수량 ÷ 경과일수
  권장입고 = (일평균 × 30) - 마켓재고 - 입고중

권장입고 ≤ 0 이면 "재고 충분" 표시
```

### 4.4 재고관리(주문) 탭

**탭 구성 (9단계)**

```
[입고권장] [발주요청*] [발주금액확정*] [구매확정] [구매완료]
[중국창고도착] [중국창고출고] [한국입항(입고중)] [상품수령완료] [전체]

* Phase 2 — 배대지 연동 후 활성화
```

**각 탭 이동 규칙**

| 현재 탭 | 이동 조건 | 다음 탭 |
|---------|----------|---------|
| 입고권장 | 재고주문 버튼 클릭 | 구매확정 |
| 구매확정 | 구매완료일 입력 또는 체크→이동 | 구매완료 |
| 구매완료 | 체크→이동 버튼 | 중국창고도착 |
| 중국창고도착 | 체크→이동 버튼 | 중국창고출고 |
| 중국창고출고 | 체크→이동 버튼 | 한국입항 |
| 한국입항 | 체크→이동 버튼 | 상품수령완료 |
| 상품수령완료 | 수량 입력 | 재고 반영 완료 |

**목록 공통 컬럼**

| 컬럼 | 설명 |
|------|------|
| 체크박스 | 다중 선택 |

| 구매요청일 | 날짜 |
| 현재상태 | 탭 상태 |
| 발주번호 | 자동 생성 |
| 스토어 | 연동 쇼핑몰 |
| 판매방식 | 로켓/그로스/윙 |
| 이미지 | 상품 이미지 |
| 상품명 | |
| 바코드 | |
| 상품링크 | |
| 발주량 | 주문 수량 |
| 구매단가 | 원가 |
| 결제총금액 | 발주량 × 단가 |
| 공장출고예정일 | 수동 |
| 재고도착예정일 | 수동 |
| 메모 | 자유 입력 |

**액션 버튼**

## 8. 전체 화면 플로우

```
[대시보드 캘린더]
  ├── 입고권장 N건 → [재고관리(주문)] > 입고권장 탭
  ├── 입고중 N건   → [재고관리(주문)] > 한국입항 탭
  └── 매출 N원     → [주문목록]

[상품관리]
  ├── 원가 이력 관리
  ├── 마켓재고 팝업
  └── 바코드 인쇄

[재고관리 - 일일매출/재고]
  ├── 스크래핑 데이터 자동 갱신
  ├── 입고권장 자동 계산
  └── 재고주문 버튼 → [재고관리(주문)] 구매확정 생성

[재고관리(주문)]
  입고권장 → 구매확정 → 구매완료
  → 중국창고도착 → 중국창고출고
  → 한국입항(입고중) → 상품수령완료
  → 재고 자동 반영
```
