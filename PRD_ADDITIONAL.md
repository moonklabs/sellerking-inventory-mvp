## 5. 재고 현황 대시보드 `/inventory/status`

### 목적
한 눈에 재고 위험 상품 파악 + 발주 필요 상품 즉시 확인

### 화면 구성

#### 5.1 상단 요약 카드 (3개)
| 카드 | 내용 |
|------|------|
| 🔴 발주 필요 | 입고권장 있는 상품 수 |
| 🟡 재고 주의 | 잔여 판매 가능일 7일 이하 |
| 🟢 재고 정상 | 정상 재고 상품 수 |

#### 5.2 재고 현황 테이블
| 컬럼 | 설명 |
|------|------|
| 상품명 | 별칭 + 상품명 |
| 국내 총재고 | 수동 입력값 |
| 마켓재고 (그로스) | 스크래핑 자동 수집 |
| 마켓재고 (로켓) | 스크래핑 자동 수집 |
| 월 목표 | 직접 설정값 |
| 일 평균 판매 | 월판매수량 / 경과일 |
| 잔여 판매 가능일 | 마켓재고 / 일평균 |
| 위험도 | 🔴🟡🟢 자동 표시 |
| 입고권장 | 계산값 |
| 바로 발주 | 버튼 |

#### 5.3 위험도 기준
- 🔴 위험: 잔여 7일 이하 or 마켓재고 0
- 🟡 주의: 잔여 8~14일
- 🟢 정상: 잔여 15일 이상

## 6. 입고 이력 관리 `/inventory/history`

### 목적
입고될 때마다 날짜/수량/원가 기록 → 원가 변동 이력 추적

### 화면 구성

#### 6.1 상품 선택 탭
- 좌측: 상품 리스트 (별칭)
- 우측: 선택된 상품의 입고 이력

#### 6.2 입고 이력 타임라인
```
2026-03-01  +960개   @3,500원   국내총재고: 1,060개   [관부가세 포함]
2026-01-15  +500개   @3,200원   국내총재고: 500개     [관부가세 포함]
             ↓ + 새 입고 추가
```

#### 6.3 신규 입고 등록 팝업
| 필드 | 타입 | 설명 |
|------|------|------|
| 입고일 | 날짜 | 오늘 기본 |
| 입고수량 | 숫자 | 필수 |
| 단가 (원화) | 숫자 | 중국 구매가 |
| 배송비 배분액 | 숫자 | LCL 계산기 연동 가능 |
| 관부가세 배분액 | 숫자 | LCL 계산기 연동 가능 |
| 실제 원가 | 자동계산 | 단가 + 배송비 + 관부가세 |
| 메모 | 텍스트 | 선택 |
| 입고 후 총재고 | 자동계산 | 기존 + 입고수량 |

#### 6.4 입고 완료 시 자동 처리
- 국내 총재고 자동 증가
- 상품관리 원가 이력 자동 업데이트
- 재고관리(주문) → 상품수령완료 탭에서 연동 가능

## 7. 월 목표 설정 `/inventory/goal`

### 목적
시즌/전략에 따라 상품별 월 판매 목표 조정 → 입고권장 재계산

### 화면 구성

#### 7.1 상품 목표 설정 테이블
| 컬럼 | 설명 |
|------|------|
| 상품명 | 별칭 + 상품명 |
| 시즌 상태 | 🌸 시즌 ON / ❄️ 시즌 OFF 토글 |
| 지난달 실판매 | 참고용 (자동) |
| 월 목표 수량 | **직접 입력** |
| 목표 변경일 | 마지막 수정일 |
| 예상 입고권장 | 목표 기반 미리보기 |
| 적용 | 저장 버튼 |

#### 7.2 입고권장 계산 로직
```
일 목표 = 월 목표 / 해당월 일수
필요 재고 = 일 목표 × 리드타임(기본 45일)
입고권장 = 필요 재고 - 현재 국내 총재고 - 입고중 수량
(음수이면 "재고 충분" 표시)
```

#### 7.3 시즌 상품 처리
- 시즌 OFF: 목표 수량 최소화 권장 알림
- 시즌 ON: 이전 시즌 실판매 데이터 참고 제안

## 8. LCL 원가 배분 계산기 `/inventory/lcl-calc`

### 목적
여러 상품 한 번에 입고 시 관부가세/배송비 자동 배분 → 정확한 원가 산출

### 배경
> "여러 개 한 번에 보내면 관부가세/배송비를 어떻게 쪼개야 할지 GPT한테 물어봤는데 세 가지가 있더라"

### 화면 구성

#### 8.1 배분 방식 선택
| 방식 | 설명 | 추천 |
|------|------|------|
| N분의 1 | 상품 개수로 균등 배분 | 단가 비슷할 때 |
| 금액 비례 | 상품 구매가 비율로 배분 | **GPT 추천 방식** |
| 수량 비례 | 입고 수량 비율로 배분 | 수량 차이 클 때 |

#### 8.2 입고 상품 목록 입력
| 컬럼 | 설명 |
|------|------|
| 상품명 | 선택 또는 직접 입력 |
| 입고수량 | 숫자 |
| 단가 (중국 구매가) | 숫자 |
| 소계 | 자동 계산 |

#### 8.3 공통 비용 입력
- 총 배송비 (중국→한국)
- 총 관부가세
- 기타 비용

#### 8.4 배분 결과 테이블
| 컬럼 | 자동계산 |
|------|---------|
| 상품별 배송비 배분 | 선택 방식 적용 |
| 상품별 관부가세 배분 | 선택 방식 적용 |
| 개당 실제 원가 | 단가 + 배분된 비용 / 수량 |
| 입고 이력에 적용 버튼 | 계산 결과 → 입고 이력 자동 입력 |

## 9. 마켓별 재고 입력 `/inventory/market-send`

### 목적
국내 총재고에서 그로스/로켓 각각 발송 수량 입력 → 마켓재고 관리

### 배경
> "그로스 100개 보내면 국내 총재고에서 100 빼고 마켓재고에 100 넣는 거죠"

### 화면 구성

#### 9.1 마켓 발송 테이블
| 컬럼 | 설명 |
|------|------|
| 상품명 | 별칭 + 옵션 |
| 국내 총재고 | 현재값 |
| 그로스 현재 재고 | 스크래핑 값 |
| 로켓 현재 재고 | 스크래핑 값 |
| 그로스 발송 수량 | **직접 입력** |
| 로켓 발송 수량 | **직접 입력** |
| 발송 후 총재고 | 자동 계산 |
| 발송 완료 | 버튼 |

#### 9.2 발송 완료 처리
- 국내 총재고 자동 차감
- 발송 이력 기록 (일시, 수량, 마켓)
- 입고권장 재계산 트리거

#### 9.3 발송 이력
- 날짜별 발송 이력 타임라인
- 마켓별 누적 발송량
